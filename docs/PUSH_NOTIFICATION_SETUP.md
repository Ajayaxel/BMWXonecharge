## A4. Push Notification Service ✅

**File:** `lib/core/services/push_notification_service.dart`

### What it does:

| Feature | How it works |
|---------|-------------|
| **Background handler** | `firebaseMessagingBackgroundHandler()` — top-level function handles messages when app is in background/terminated |
| **Permission request** | Requests notification permission on iOS & Android 13+ |
| **Foreground notifications** | Listens via `FirebaseMessaging.onMessage` → displays as local notification banner using `flutter_local_notifications` |
| **Background tap** | `FirebaseMessaging.onMessageOpenedApp` — handles when user taps notification while app is in background |
| **Terminated tap** | `getInitialMessage()` — handles when app was opened from a terminated state via notification tap |
| **FCM Token** | Gets device token and sends to PHP backend via `POST /api/customer/fcm-token` |
| **Token refresh** | Listens for token changes and re-sends to backend |
| **iOS Simulator safety** | Checks APNS token availability before requesting FCM token (avoids crash on simulator) |

### Key Methods:

```dart
// Auto-called on app start
await pushNotificationService.initialize(storage: secureStorage);

// Call after user logs in (to send FCM token with auth)
await PushNotificationService().sendTokenAfterLogin(secureStorage);

// Subscribe to topics (optional)
await PushNotificationService().subscribeToTopic('promotions');
```

### FCM Token → Backend API Call:

The service automatically sends the FCM token to your backend:

```
POST https://app.onecharge.io/api/customer/fcm-token
Headers: Authorization: Bearer <user_auth_token>
Body: {
  "fcm_token": "dK8x...long-token...F2a",
  "device_type": "ios"  // or "android"
}
```

### Notification Tap Navigation:

When a user taps a notification, it checks the `type` field in the data payload:

```dart

  case 'booking_success':
```

## B2. Create Database Table

Run this SQL on your database:

```sql
CREATE TABLE customer_fcm_tokens (
    id INT AUTO_INCREMENT PRIMARY KEY,
    customer_id INT NOT NULL,
    fcm_token VARCHAR(500) NOT NULL,
    device_type ENUM('ios', 'android') NOT NULL DEFAULT 'android',
    is_active TINYINT(1) DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_customer_id (customer_id),
    UNIQUE KEY unique_token (fcm_token)
);
```

**Why this table?**
- One customer can have **multiple devices** (phone + tablet) → multiple FCM tokens
- `is_active` = 0 when a token becomes invalid (user uninstalled app, etc.)
- `UNIQUE KEY` on `fcm_token` prevents duplicates

### If using Laravel:

```php
// routes/api.php

use Illuminate\Http\Request;
use Illuminate\Support\Facades\DB;

Route::middleware('auth:sanctum')->post('/customer/fcm-token', function (Request $request) {

        'fcm_token' => 'required|string|max:500',
        'device_type' => 'required|in:ios,android',

    DB::table('customer_fcm_tokens')->updateOrInsert(
        ['fcm_token' => $request->fcm_token],  // Match by token
        [
            'customer_id' => $customerId,
            'device_type' => $request->device_type,
            'is_active' => 1,
            'updated_at' => now(),
        ]
    );

    return response()->json([
        'success' => true,
        'message' => 'FCM token saved successfully',
    ]);
});
```

## B5. Create FCM Notification Service Class

This is the core class that sends push notifications via Firebase Cloud Messaging HTTP v1 API.

### If using Laravel:

```php
<?php
// app/Services/FCMService.php

namespace App\Services;

use Google\Client as GoogleClient;
use Illuminate\Support\Facades\DB;
use Illuminate\Support\Facades\Log;

class FCMService
{
    private string $projectId = 'onecharge-26995';
    private string $serviceAccountPath;

    public function __construct()
    {
        $this->serviceAccountPath = base_path('firebase-service-account.json');
    }

    /**
     * ──────────────────────────────────────────────
     * Get OAuth2 access token for FCM API
     * ──────────────────────────────────────────────
     */
    private function getAccessToken(): string
    {
        $client = new GoogleClient();
        $client->setAuthConfig($this->serviceAccountPath);
        $client->addScope('https://www.googleapis.com/auth/firebase.messaging');

        $token = $client->fetchAccessTokenWithAssertion();

        if (isset($token['error'])) {
            Log::error('FCM Auth Error: ' . json_encode($token));
            throw new \Exception('Failed to get FCM access token');
        }

        return $token['access_token'];
    }

    /**
     * ──────────────────────────────────────────────
     * Send notification to a SINGLE device
     * ──────────────────────────────────────────────
     *
     * @param string $fcmToken  Device FCM token
     * @param string $title     Notification title
     * @param string $body      Notification body text
     * @param array  $data      Extra data payload (key-value pairs, values MUST be strings)
     */
    public function sendToDevice(
        string $fcmToken,
        string $title,
        string $body,
        array $data = []
    ): array {
        $accessToken = $this->getAccessToken();

        $url = "https://fcm.googleapis.com/v1/projects/{$this->projectId}/messages:send";

        // Build the FCM message
        $message = [
            'message' => [
                'token' => $fcmToken,

                // Notification (shown as banner)
                'notification' => [
                    'title' => $title,
                    'body' => $body,
                ],

                // Data payload (received by app for logic/navigation)
                'data' => array_map('strval', $data),  // All values MUST be strings

                // Android-specific settings
                'android' => [
                    'priority' => 'high',
                    'notification' => [
                        'channel_id' => 'onecharge_notifications',
                        'sound' => 'default',
                        'default_vibrate_timings' => true,
                    ],
                ],

                // iOS (APNs) specific settings
                'apns' => [
                    'payload' => [
                        'aps' => [
                            'alert' => [
                                'title' => $title,
                                'body' => $body,
                            ],
                            'sound' => 'default',
                            'badge' => 1,
                            'content-available' => 1,
                        ],
                    ],
                    'headers' => [
                        'apns-priority' => '10',
                    ],
                ],
            ],
        ];

        // Send via cURL
        $ch = curl_init($url);
        curl_setopt_array($ch, [
            CURLOPT_POST => true,
            CURLOPT_HTTPHEADER => [
                'Authorization: Bearer ' . $accessToken,
                'Content-Type: application/json',
            ],
            CURLOPT_POSTFIELDS => json_encode($message),
            CURLOPT_RETURNTRANSFER => true,
            CURLOPT_TIMEOUT => 30,
        ]);

        $response = curl_exec($ch);
        $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
        $error = curl_error($ch);
        curl_close($ch);

        if ($error) {
            Log::error("FCM cURL Error: $error");
            return ['success' => false, 'error' => $error];
        }

        $result = json_decode($response, true);

        if ($httpCode !== 200) {
            Log::error("FCM Error [$httpCode]: " . $response);
        } else {
            Log::info("FCM sent to device: $title");
        }

        return [
            'success' => $httpCode === 200,
            'http_code' => $httpCode,
            'response' => $result,
        ];
    }

    /**
     * ──────────────────────────────────────────────
     * Send notification to ALL devices of a customer
     * ──────────────────────────────────────────────
     *
     * One customer can have multiple devices (phone + tablet).
     * This sends to all of them.
     */
    public function sendToCustomer(
        int $customerId,
        string $title,
        string $body,
        array $data = []
    ): void {
        // Get all active FCM tokens for this customer
        $tokens = DB::table('customer_fcm_tokens')
            ->where('customer_id', $customerId)
            ->where('is_active', 1)
            ->pluck('fcm_token');

        if ($tokens->isEmpty()) {
            Log::warning("No FCM tokens found for customer: $customerId");
            return;
        }

        foreach ($tokens as $token) {
            $result = $this->sendToDevice($token, $title, $body, $data);

            // If token is invalid, mark it as inactive in the database
            if (!$result['success'] && isset($result['response']['error'])) {
                $errorCode = $result['response']['error']['details'][0]['errorCode'] ?? '';
                if (in_array($errorCode, ['UNREGISTERED', 'INVALID_ARGUMENT'])) {
                    DB::table('customer_fcm_tokens')
                        ->where('fcm_token', $token)
                        ->update(['is_active' => 0]);

                    Log::info("Deactivated invalid FCM token for customer: $customerId");
                }
            }
        }
    }
}
```

```php
        ];
    }

    public function sendToCustomer(int $customerId, string $title, string $body, array $data = []): void
    {
        $stmt = $this->pdo->prepare("
            SELECT fcm_token FROM customer_fcm_tokens
            WHERE customer_id = :customer_id AND is_active = 1
```

# PART D: Notification Types Reference

## Data Payload Structure

Every notification sent from PHP should include a `type` field in the `data` payload. This tells the Flutter app **what to do** when the notification is tapped.

| Type | When to Send | Data Fields |
|------|-------------|-------------|
| `booking_success` | Booking confirmed / completed | `booking_id`, `service_type`, `status` |
| `driver_status` | Driver accepted / on the way / arrived | `booking_id`, `status`, `driver_name`, `driver_lat`, `driver_lng` |

### Driver Status Values:

| Status | Meaning |
|--------|---------|
| `accepted` | Driver accepted the request |
| `on_the_way` | Driver is driving to customer |
| `arrived` | Driver has arrived at location |
| `in_progress` | Service is in progress |
| `completed` | Service is completed |

```
     ↑ App stays open, user can tap banner
```

- **How:** `FirebaseMessaging.onMessage` → `flutter_local_notifications` shows banner

```
┌─────────────────────────────────────────┐
```

- **Tap:** App launches → `getInitialMessage()` fires → navigates to booking/tracking screen

## Driver Location in Notifications — How It Works

> **Q: If the app is closed, will the driver location show on the notification?**

**The notification shows TEXT only** (title + body). It does NOT show a map or live location.

However, the **driver's lat/lng is hidden inside the `data` payload**. When the user taps the notification, the app opens and uses that data to show the driver on a map.

### Visual Flow:

```
PHP Backend sends notification:
┌──────────────────────────────────────────────────────┐
│  notification: {                                      │
│    title: "🚗 Driver On The Way"          ← VISIBLE  │
│    body: "John is 5 mins away!"           ← VISIBLE  │
│  }                                                    │
│  data: {                                              │
│    type: "driver_status"                  ← HIDDEN    │
│    booking_id: "123"                      ← HIDDEN    │
│    driver_lat: "25.1972"                  ← HIDDEN    │
│    driver_lng: "55.2744"                  ← HIDDEN    │
│  }                                                    │
└──────────────────────────────────────────────────────┘
           │
           ▼
User sees notification banner:
┌─────────────────────────────────┐
│ 🚗 Driver On The Way            │  ← Only title + body
│ John is 5 mins away!            │  ← No map, no location
└─────────────────────────────────┘
           │
      User taps
           │
           ▼
App opens → reads data payload:
  { driver_lat: "25.1972", driver_lng: "55.2744", booking_id: "123" }
           │
           ▼
Navigates to Tracking Screen:
┌─────────────────────────────────┐
│  ┌─────────────────────────┐    │
│  │     Google Map           │    │
│  │                          │    │
│  │    🚗 ← Driver marker   │    │
│  │     |                    │    │
│  │     | (polyline)         │    │
│  │     |                    │    │
│  │    📍 ← Customer         │    │
│  │                          │    │
│  └─────────────────────────┘    │
│                                  │
│  Driver: John                    │
│  ETA: 5 minutes                  │
└─────────────────────────────────┘
```

### Summary Table:

| App State | Notification Banner? | Live Map? | Driver Location? |
|-----------|:---:|:---:|:---:|
| **App Open (foreground)** | ✅ Banner at top | ✅ Already visible (if on tracking screen) | ✅ Via socket/API |
| **App Minimized (background)** | ✅ System notification | ❌ Not until user taps | ✅ After tap → opens map |
| **App Closed (terminated)** | ✅ System notification | ❌ Not until user taps | ✅ After tap → opens map |

### Important Notes:

1. **Push notifications are text-only** — they cannot show maps, images, or live location
2. **Driver location data is sent in the `data` payload** — invisible to user but used by the app
3. **Live tracking requires the app to be open** — with an active socket connection for real-time updates
4. **The notification gets the driver to open the app** → then the app handles showing the live map
